What You're Building

A production-grade static website deployment using core AWS services — the same pattern used by real-world web applications.

  • ▸S3 bucket — private, versioned, with access logging to a separate log bucket
  • ▸CloudFront distribution — HTTPS enforced, HTTP/2, PriceClass_100 (US/EU/Asia edge locations)
  • ▸Origin Access Control (OAC) — SigV4-signed requests; S3 bucket never directly accessible
  • ▸S3 bucket policy — resource-based policy scoped to this specific CloudFront distribution ARN
  • ▸CloudWatch alarm — alerts when 4xx error rate exceeds 5% for 10 minutes

🧠 Key Concepts Practiced

Security & Access Control

  • ✓OAC vs OAI — OAC is the modern replacement for Origin Access Identity. Uses SigV4 request signing and supports SSE-KMS encrypted buckets.
  • ✓S3 Block Public Access — Four separate controls: BlockPublicAcls, IgnorePublicAcls, BlockPublicPolicy, RestrictPublicBuckets. All enabled.
  • ✓Bucket Policy Conditions — AWS:SourceArn condition restricts access to one specific CloudFront distribution. Not just any CloudFront — this one.
  • ✓HTTPS Enforcement — ViewerProtocolPolicy: redirect-to-https sends all HTTP traffic to HTTPS automatically.

Storage & Delivery

  • ✓S3 Versioning — Every PUT creates a new version. Protects against accidental deletes and overwrites. Versions are listed with unique version IDs.
  • ✓S3 Access Logging — Logs every request to the website bucket into a separate log bucket with a prefix. Important for audit and debugging.
  • ✓Lifecycle Policies — Automates log file tiering: Standard → Standard-IA (30d) → Glacier (90d) → Delete (365d). Reduces storage cost.
  • ✓CloudFront Caching — Managed cache policy 658327ea (CachingOptimized) used for best-practice S3 caching with Gzip and Brotli compression.

Observability & Tagging

  • ✓CloudWatch Alarm States — Three states: OK, ALARM, INSUFFICIENT_DATA. Missing data is treated as "not breaching" to avoid false alarms.
  • ✓CloudFront Metrics — Published to CloudWatch automatically in the AWS/CloudFront namespace with Global region dimension.
  • ✓Resource Tagging — All resources tagged with Project, Environment, ManagedBy. Enables cost allocation and filtering via AWS Resource Groups.
  • ✓IaC via Shell Script — Full deploy/cleanup automation using AWS CLI. Every resource ID saved to .env.project for reliable teardown.

Cleanup & Cost Control

  • ✓Correct teardown order — CloudFront must be disabled before deletion. S3 buckets must be emptied (including versions) before deletion.
  • ✓OAC cleanup — OAC must be detached from distributions before it can be deleted. cleanup.sh handles this automatically.
  • ✓Tag-based verification — After cleanup, aws resourcegroupstaggingapi get-resources confirms zero tagged resources remain.
  • ✓Free tier awareness — S3 (5GB/20k GETs), CloudFront (1TB/10M requests), CloudWatch (10 alarms) all stay within free tier for this lab.

💻 Commands Reference

Deploy

# 1. Configure AWS CLI (one-time)
aws configure

# 2. Verify credentials
aws sts get-caller-identity

# 3. Run deploy script
bash ~/workspace/ai-terminal-workflow/ \
  projects/01-static-website-aws-fundamentals/ \
  scripts/deploy.sh

Cleanup (always run after lab)

# Load saved resource IDs
source ~/workspace/ai-terminal-workflow/ \
  projects/01-static-website-aws-fundamentals/ \
  .env.project

# Run cleanup script
bash ~/workspace/ai-terminal-workflow/ \
  projects/01-static-website-aws-fundamentals/ \
  scripts/cleanup.sh

# Verify nothing remains
aws resourcegroupstaggingapi get-resources \
  --tag-filters Key=Project,Values=aws-cert-study

Useful AWS CLI Checks

# Check CloudFront distribution status
aws cloudfront get-distribution \
  --id $DISTRIBUTION_ID \
  --query 'Distribution.Status'

# List S3 object versions
aws s3api list-object-versions \
  --bucket $BUCKET_NAME

# Check CloudWatch alarm state
aws cloudwatch describe-alarms \
  --alarm-names "CloudFront-4xx-Errors-High"

Test Security (expect 403)

# Direct S3 access should return 403
# (blocked by bucket policy — good!)
curl -I https://$BUCKET_NAME.s3.amazonaws.com/index.html

# CloudFront access should return 200
curl -I https://$CF_DOMAIN/index.html

# HTTP should redirect to HTTPS (301)
curl -I http://$CF_DOMAIN/index.html
