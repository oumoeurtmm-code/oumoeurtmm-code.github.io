Workflow 1 — New Hire Provisioning

flowchart TD A([HRIS Webhook]) --> B[Normalize Payload] B --> C[OpenCode\nCoordinatorAgent] C --> D{Valid JSON Plan?} D -- No --> E[Flag: Manual Review\nSlack #it-alerts] D -- Yes --> F{High-Risk Role?} F -- Yes --> G[Send Approval\nSlack #it-approvals] G --> H[Wait for Approval] H --> I F -- No --> I[Loop: Apply\nOkta Actions] I --> J[Loop: Apply\nEntra Actions] J --> K[OpenCode\nRunbookAgent] K --> L[Notify: Complete\nSlack #it-ops] style A fill:#1e3a5f,stroke:#60a5fa,color:#e2e8f0 style C fill:#0c3a4a,stroke:#22d3ee,color:#e2e8f0 style K fill:#0c3a4a,stroke:#22d3ee,color:#e2e8f0 style E fill:#3d0000,stroke:#f87171,color:#e2e8f0 style L fill:#052e16,stroke:#34d399,color:#e2e8f0

Step-by-Step

1
HRIS Webhook
Receives new hire payload from Workday, BambooHR, or any HRIS
2
Normalize Payload
Maps HRIS fields → structured newHireEvent JSON with cloudProfile
3
CoordinatorAgent (OpenCode)
Gemini/Perplexity decide entitlements. Claude Code generates any needed scripts. Returns strict JSON plan.
4
Validate + Risk Gate
JSON schema check → if high-risk role, require manager approval via Slack
5
Apply Okta + Entra
Loop over oktaActions[] and entraActions[] via native n8n nodes
6
RunbookAgent
Auto-generates Markdown runbook with verification and rollback steps

Workflow 2 — Access Request Decision

flowchart TD A([Slack Command\nor ITSM Ticket]) --> B[Extract Request Context] B --> C[OpenCode\nPolicyAgent] C --> D[Parse Decision JSON] D --> E{Route Decision} E -- auto_grant --> F[Apply Access\nOkta + Entra] F --> G[Notify: Granted ✅] E -- needs_approval --> H[Send to Manager\nSlack] H --> I[Wait for Decision] I --> J[Apply Approved Access] E -- deny --> K[Notify: Denied ❌\nwith reason] G --> L[RunbookAgent\nAudit Log] J --> L K --> L style A fill:#1e3a5f,stroke:#60a5fa,color:#e2e8f0 style C fill:#0c3a4a,stroke:#22d3ee,color:#e2e8f0 style L fill:#0c3a4a,stroke:#22d3ee,color:#e2e8f0 style G fill:#052e16,stroke:#34d399,color:#e2e8f0 style K fill:#3d0000,stroke:#f87171,color:#e2e8f0

Decision Logic

auto_grant
Low-risk, standard access matching user role. Applied immediately via Okta/Entra nodes.
needs_approval
Elevated permissions or cross-team access. Routes to manager via Slack with AI reasoning included.
deny
Violates least-privilege or zero-trust policy. Requester notified with specific reason from PolicyAgent.
